Performance Optimization in Node.js
* a. Avoid Blocking the Event Loop
Node.js is single-threaded, so any blocking code will slow down your app.

Don’t use synchronous functions like fs.readFileSync(), crypto.pbkdf2Sync(), etc.

Use setImmediate(), process.nextTick(), and worker_threads for heavy computation.

📌 Interview Tip: Explain how blocking the event loop affects all incoming requests.

* b. Use Asynchronous I/O
Always prefer non-blocking I/O operations:
// Good
fs.readFile('file.txt', (err, data) => {
  // handle response
});
* c. Enable HTTP Keep-Alive
Avoid TCP handshake overhead by enabling Keep-Alive:
const server = http.createServer(app);
server.keepAliveTimeout = 60 * 1000;
* d. Use Gzip/Compression
Compress responses using compression middleware:
npm install compression
const compression = require('compression');
app.use(compression());
* e. Optimize Middleware and Routing
Only use required middleware for each route

Order routes efficiently (frequently-used routes should come first)

* f. Use Caching (Redis, memory-cache)
Reduce load on your app and database using caching.

2. Throughput Optimization
✅ a. Use Cluster Module
Leverage all CPU cores using cluster:
const cluster = require('cluster');
const os = require('os');

if (cluster.isMaster) {
  for (let i = 0; i < os.cpus().length; i++) cluster.fork();
} else {
  // Start server
}

* b. Load Balancing
Use Nginx or a cloud load balancer in front of your Node.js processes.

* c. Efficient DB Queries
Use indexes

Avoid SELECT *

Use connection pools (pg, mongoose, etc.)

Use batching/bulk inserts

* d. Handle Backpressure in Streams
When dealing with high throughput (like file uploads), manage backpressure using stream .pause() and .resume().
