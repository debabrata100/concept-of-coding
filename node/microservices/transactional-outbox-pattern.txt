1. ✅ Transactional Outbox Pattern
🔍 What It Is:
A way to reliably publish events/messages after a database update, ensuring that DB writes and event publication are atomic (even if the broker is down).

🧠 Why It Matters:
Prevents data loss or duplication during failures.

Ensures eventual consistency between services.

💡 Implementation in MongoDB:
MongoDB doesn't support multi-resource transactions (e.g., Mongo + Kafka), so you:

Write your event to a collection (outbox) as part of your DB transaction.

Run a background worker to poll unprocessed outbox messages and publish them to Kafka/RabbitMQ.

// 1. Insert to "orders" and "outbox" in one atomic transaction
const session = await mongoose.startSession();
await session.withTransaction(async () => {
  await Order.create([{ id: '123', status: 'created' }], { session });
  await Outbox.create([{
    eventType: 'OrderCreated',
    payload: { orderId: '123' },
    published: false
  }], { session });
});
session.endSession();

// 2. Background worker polls the outbox
const outboxEvents = await Outbox.find({ published: false }).limit(10);
for (const event of outboxEvents) {
  await kafkaProducer.send({ topic: 'orders', message: event.payload });
  await Outbox.updateOne({ _id: event._id }, { published: true });
}

✅ 2. Deduplication
Even if an event is accidentally sent to Kafka more than once (e.g., due to retries or worker crashes), consumers should be able to ignore duplicates.

💡 How?
Include a unique event ID in the Kafka message, and ensure idempotency on the consumer side.

Outbox schema with eventId:

3. Retry with Exponential Backoff
If Kafka is temporarily unavailable, you should retry with backoff and possibly move the event to a Dead Letter Queue (DLQ) after N failures.

💡 Add retry metadata:
Extend your Outbox schema:

{
    eventId: String,
    eventType: String,
    payload: Object,
    published: Boolean,
    retryCount: { type: Number, default: 0 },
    lastTriedAt: Date,
    error: String
  }
  Modified worker logic with retries:
  
  const outboxEvents = await Outbox.find({ published: false }).limit(10);

for (const event of outboxEvents) {
  try {
    await kafkaProducer.send({
      topic: 'orders',
      message: { key: event.eventId, value: JSON.stringify(event.payload) }
    });

    await Outbox.updateOne({ _id: event._id }, { published: true, error: null });
  } catch (err) {
    const retryLimit = 5;

    if (event.retryCount + 1 >= retryLimit) {
      console.error(`Event ${event._id} failed too many times. Moving to DLQ.`);
      // Optionally, move to a DLQ collection
    }

    await Outbox.updateOne(
      { _id: event._id },
      {
        $inc: { retryCount: 1 },
        $set: {
          lastTriedAt: new Date(),
          error: err.message
        }
      }
    );
  }
}
