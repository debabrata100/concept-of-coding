What is immutable update ?
https://chatgpt.com/share/9302b375-9cc7-4dc9-afc1-2a38954a899d
In React, an immutable update refers to updating the state in a way that does not mutate (change) the original state object. Instead, a new state object is created with the desired updates. This practice is important because it ensures that React can correctly detect changes to the state and efficiently update the user interface.

Here's why immutable updates are crucial in React:

Efficient Re-rendering: React relies on a virtual DOM to determine what parts of the actual DOM need to be updated. If state changes are done immutably, React can quickly compare the previous and new state objects (using shallow comparison) to see what has changed, leading to efficient re-rendering.

Predictability and Debugging: Immutable updates ensure that previous state objects are not altered, which makes the application more predictable and easier to debug. You can trace state changes over time without worrying about unintended side effects from mutated state objects.

Pure Components: Immutable updates align well with the concept of pure components in React. A pure component re-renders only when its props or state changes. Immutable updates help ensure that these changes are detected accurately.
